# AEGIS-SE Reports Directory

This directory contains generated compliance and analysis reports for the AEGIS-SE defense systems project.

## 📁 Directory Structure

```
reports/
├── compliance/          # Compliance reports (generated, not committed)
│   ├── misra/          # MISRA C:2012 static analysis reports
│   ├── security/       # Security compliance assessments
│   └── do178c/         # DO-178C certification readiness reports
└── .gitkeep           # Preserve directory structure
```

## ⚠️ Important Notes

### **Generated Reports (NOT in Git)**
- `misra_report_*.xml` - Static analysis results from cppcheck
- `security_report_*.txt` - Security compliance status summaries
- `do178c_report_*.txt` - Certification readiness assessments
- All reports include timestamps and are regenerated on each CI/CD run

### **Report Generation**
Reports are automatically generated by:
- **CI/CD Pipeline**: GitHub Actions workflow generates reports on every build
- **Manual Generation**: Run `scripts/generate_compliance_report.sh`
- **Build System**: `make compliance-report` target

### **Report Contents**

#### MISRA C Reports
- Static analysis violations and compliance status
- Code quality metrics and recommendations
- Safety-critical code compliance verification

#### Security Reports
- FIPS 140-2 Level 3 compliance status
- Common Criteria EAL4+ readiness assessment
- Security control implementation status

#### DO-178C Reports
- Airborne software certification readiness
- Code coverage analysis for safety-critical functions
- Formal verification and testing compliance

## 🔒 Security Classification

**Classification**: UNCLASSIFIED
**Distribution**: Approved for public release (generated reports may contain sensitive technical details)

## 📊 Accessing Reports

### In CI/CD
Reports are available as GitHub Actions artifacts:
- Navigate to Actions → Workflow Run → Artifacts section
- Download compliance-reports.zip

### Local Development
```bash
# Generate fresh reports
./scripts/generate_compliance_report.sh

# View latest reports
ls -la reports/compliance/*/
```
